编写友好的脚本函数

python function

在我使用的命令行工具中，有些程序令我非常困惑，它们没有任何的输出和提示信息，我不知道这个程序是否在正常运行，特别是在国内特殊的网络环境下。

我觉得一个友好的脚本程序是带有输出或进度提示的，特别是操作可能会花费比较多或者无法预知运算时间的，如果有一个 verbose 可选项，我想会是非常不错的注意。 例如：

def func1(verbose=0, *args, **kwargs):
    if verbose > 1:
        print('verbose greater than 1')

    if verbose > 2:
        print('verbose greater than 2')

还有一种比较常用处理方式：

def init_slug():
    aas = AdministrativeArea.objects.filter(slug__isnull=True)
    for aa in aas:
        aa.slug = slugify(aa.name)
        aa.save()

上面的代码中 AdministrativeArea 是 django 中的一个 model，代码的大致意思是将 model 中的 name 转化成 slug 并保存。通常情况下，这会花费我大量的时间。

我一般会写成这个样子

def init_slug():
    aas = AdministrativeArea.objects.filter(slug__isnull=True)
    total = aas.count()
    for index,aa in enumerate(aas):
        print("Processing aa {0}/{1}".format(index,total))
        aa.slug = slugify(aa.name)
        aa.save()

go lang 安装

go golang install ubuntu

﻿虽然我不用 golang 编程，但是有很多 go 写的程序非常不错，需要自己编译安装，所以安装 golang 是必须的。

一、下载


wget https://storage.googleapis.com/golang/go1.4.3.linux-amd64.tar.gz

二、安装


tar -C /usr/local -xzf go1.4.3.linux-amd64.tar.gz

三、加入 PATH 与 设置 GOPATH

将 go 解压到 /usr/local/go 后，需要将这个目录加到 PATH 中

修改 .bashrc 或 .zshrc ，这取决于你的的 shell


export GOPATH=/home/recall/gopath

export PATH=$GOPATH/bin:$PATH

export PATH=/usr/local/go/bin:$PATH

golang 要求必须设置 GOPATH 环境变量

  • src 存放源代码（比如：.go .c .h .s等）

  • pkg 编译后生成的文件（比如：.a）

  • bin 编译后生成的可执行文件

上面的例子中，我的用户名叫做 recall， 我在其中创建的一个文件夹叫做 gopath，然后将 GOPATH 下的 bin 目录加入到 PATH 中，这样 go 编译后的程序就能够直接执行了。

第三行是将 go 所在目录加入到PATH，这样我们就能够直接使用 go 命令了。

四、测试

安装完成后，运行


go env

可以查看当前 go 的所有环境变量。


GOARCH="amd64" 
GOBIN="" 
GOCHAR="6" 
GOEXE="" 
GOHOSTARCH="amd64" 
GOHOSTOS="linux" 
GOOS="linux" 
GOPATH="/home/recall/gopath" 
GORACE="" 
GOROOT="/usr/local/go" 
GOTOOLDIR="/usr/local/go/pkg/tool/linux_amd64" 
CC="gcc" 
GOGCCFLAGS="-fPIC -m64 -pthread -fmessage-length=0" 
CXX="g++" 
CGO_ENABLED="1"

五、备注

如果使用 go get 获取go软件包的时候出现错误


go get golang.org/x/crypto/ssh

比如上面这个，出现问题的话，可以从github 上下载，放到对应的目录上即可。

地址为 https://github.com/golang/crypto

systemd 配置 supervisor

systemd supervisor

最近升级到了 kubuntu 15.10，系统启动管理由 init 变成了 systemd。

我一般习惯使用 supervisor 来管理我的程序，但是必须要系统开机启动，下面是我的一些记录。

一、安装 supervisor

切换到 root 或使用 sudo 安装


pip install supervisor

二、默认的 supervisor 配置文件


echo_supervisord_conf > supervisord.conf

sudo mv supervisord.conf /etc/

三、测试 supervisor 能够正常启动


supervisord -c /etc/supervisord.conf

四、创建 supervisor 的 systemd 启动配置文件

在目录 /lib/systemd/system 下， 创建文件 supervisord.service

内容如下


[Unit] 
Description=supervisord - Supervisor process control system for UNIX 
Documentation=http://supervisord.org 
After=network.target 

[Service] 
Type=forking 
ExecStart=/usr/local/bin/supervisord -c /etc/supervisord.conf 
ExecReload=/usr/local/bin/supervisorctl reload 
ExecStop=/usr/local/bin/supervisorctl shutdown 

[Install] 
WantedBy=multi-user.target

需要注意的是，如果你的 supervisord 所在的目录和我的不一样，你需要先查找 supervisord 的位置，然后修改成合适的值。

五、开机启动

启动 supervisor


systemctl start  supervisord.service

如果不能正常启动，可以用 journalctl -xe 查看错误信息。

加入开机启动


systemctl enable supervisord.service

docker 阿里云加速

docker pull aliyun

安装或升级Docker

请安装1.6.0以上版本的Docker。 您可以通过阿里云的镜像仓库下载： mirrors.aliyun.com/help/docker-engine

curl -sSL http://acs-public-mirror.oss-cn-hangzhou.aliyuncs.com/docker-engine/internet | sh -

配置Docker加速器

您可以使用如下的脚本将mirror的配置添加到docker daemon的启动参数中。

echo "DOCKER_OPTS=\"--registry-mirror=https://0yjmvwvv.mirror.aliyuncs.com\"" | sudo tee -a /etc/default/docker
sudo service docker restart

备注

如果已经安装了新版本的 docker，只需要执行后面的配置就可以了
